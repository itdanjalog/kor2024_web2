1. 프로젝트 만들기 : jdk17 , gradle , groovy
2. 프로젝트 사용할 초기 라이브러리(dependencies)
    - https://start.spring.io/
3. 프로젝트 폴더 세팅
    java
        korweb(최상위패키지)              : 임의 패키지명 생성 , 주로 홈페이지명으로 한다.
            AppStart클래스               : 스프링 시작 클래스
            controller패키지             : HTTP 매핑 관련 클래스들 패키지
            model패키지
                - entity 패키지          : DB 매핑 관련 클래스들
                - dto 패키지             : dto 데이터 이동객체 클래스들
                - repository패키지       : entity 조작 인터페이스 들
            service패키지                : 자바의 비지니스 로직 처리하는 클래스들
    resources
        application.properties          : 스프링 설정 파일.
        static 패키지                    : 정적파일( css,js,img 등등 )
            css 패키지
            js 패키지
            img 패키지
            favicon.ico
        templates 패키지                 : 동적파일( 타임리프 , JSP 등등 )

4. 기본적인 application.properties 준비
    1. server.port : spring web tomcat 서버의 port 변경
    2. JDBC CONNECT

세션 : 아파치 톰캣(서버) 안에 저장소
    - HTTP 요청에 따른 브라우저 마다의 저장소
    - 주로 로그인정보 , 비회원제 데이터(장바구니,결제) 보안 O
    - 생명주기 : 서버가 꺼지면 사라짐 , 또는 세션 객체 속성들을 초기화
    - 단점 : 많이 사용하면 웹 서버에 과부하.
    - 메소드
        HttpServletRequest request 객체
        1. 세션 객체( 타입 : Object ) 호출
            request.getSession()
        2. 세션 객체의 속성 추가
            request.getSession().setAttribute( "key", value );
        3. 세션 객체의 속성 호출( 캐스팅 필요 )
            request.getSession().getAttribute("key")
        4. 세션 객체의 속성들을 초기화
            request.getSession().invalidate();
            - 특정 속성 값만 초기화
            request.getSession().setAttribute( "key", null );
        5. 세션 객체의 속성들의 데이터 생명주기( 초 단위 )
            request.getSession().setMaxInactiveInterval( 생명시간 );
